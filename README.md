# Capgemini Data Engineering Training
### SQL  PySpark  ETL  Data Quality  Analytics

## About This Repository

This repository tracks a practical data engineering workshop focused on:
- Week 0: foundational ETL, SQL-to-PySpark mapping, and analytics pipeline design
- Week 1: daily hands-on SQL and PySpark practice
- Week 2: intermediate query patterns, CTEs, window functions, and pipeline refinement

The top-level folders contain the training materials, sample files, execution scripts, and outputs for each phase.

## Week 0 - Learning Pathway 1

This is the foundational week. It covers the full learning pathway from:
- SQL basics
- PySpark DataFrame operations
- ETL pipeline design
- Advanced analytics with the Olist dataset

### What to expect in week0/
- phase1/: SQL to PySpark basics
- phase2/: JOINs, aggregations, and real dataset bridging
- phase3/: ETL pipeline fundamentals
- phase3A/: data quality and cleaning challenge
- phase4/: end-to-end business pipeline with analytics reporting
- phase4A/: segmentation and bucketization methods
- phase5/: advanced analytics and Olist capstone work
- phase6/: capstone/advanced integration project

> Start with week0/README.md and follow the phases sequentially.

---

## Week 1 - Structured Practice

week1/ is organized as day-by-day practice exercises.
Each day includes code examples, SQL query files, and PySpark practice scripts.

### What to expect in week1/
- DAY1/: foundational PySpark and SQL query examples
- DAY2/: joins, analytical queries, and query output exercises
- DAY3/: window functions, ranking, and advanced SQL patterns
- DAY4/: continued SQL/PySpark practice and review
- DAY5/: applied practice using sample datasets

> Use week1/README.md to track daily progress and open the matching day folder.

---

## Week 2 - Intermediate SQL and Pipeline Patterns

week2/ continues the training with deeper SQL techniques and pipeline examples.
This week focuses on:
- CTEs and subqueries
- LAG / LEAD and window-based analytics
- multi-step transformations and cleanup patterns

### What to expect in week2/
- DAY1/: CTEs and SQL query practice
- DAY2/: window functions and advanced aggregation
- DAY3/: pipeline-style queries and validation

> Use week2/README.md for the lesson plan and task list.

---

## How to Work Through the Training

1. Open the root README.md for the overall roadmap.
2. Start in week0/ and complete the phase-based learning pathway.
3. Continue with week1/ for daily practical exercises.
4. Finish with week2/ for more advanced SQL and pipeline patterns.
5. Review outputs in each folder to compare results.
